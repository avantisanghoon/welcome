mysql 구조

관계db 는 스프레드시트와 비슷한 구조를 갖고 있다.
표 형식
표가 많아지면 정리할 필요가 있어짐

연관된 표를 그룹핑 하고 연관되지 않는 표를 구분해 준다. 이를 데이터베이스라 한다. 표를 그룹핑 하는것
스키마 = 연관된 표들을 그룹핑 할때 사용하는 폴더
스키마가 많아지면 저정할 공간이 필요 = 데이터베이스 서버

SQL
STRUCTURED : 표, 정리정돈, 구주화된 
QUERY : 요청, 질의 (데이터베이스에게)
LANGUAGE : 공통의 약속에 따른 커뮤니케이션 언어. 데이터베이스서버와 사용자간
1. 어떤 컴퓨터 언어보다 쉽다
2. 중요하다.
3. 관계형db의 모든 제품군이 사용하는 표준화된 언어이다.
4. 가성비가 뛰어난 언어이다.

DB를 잘 다룬다는 말은
필요와 상황에 따라서 원하하는 바를 "SELECT" 를 잘 다룬다는 것 이다.
